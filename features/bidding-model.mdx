# Bidding request data model

The `zonaweb.bidding.request` model stores all submitted project proposals and their workflow state.

## Model metadata

| Property | Value |
|---|---|
| `_name` | `zonaweb.bidding.request` |
| `_description` | Solicitud de Licitación Zonaweb |
| `_inherit` | `['mail.thread', 'mail.activity.mixin']` |
| `_order` | `create_date desc` (newest first) |

The model inherits `mail.thread` and `mail.activity.mixin`, enabling the Odoo chatter panel, message logging, and scheduled activity support in backend views.

## Fields

### Identity fields

<ParamField path="name" type="Char">
  Auto-generated reference number in the format `BID/XXXXX` (e.g., `BID/00001`). Set to `readonly=True` after creation. The default value is the translated string `'Nuevo'` until the `create` override assigns the real sequence value. `copy=False` ensures duplicated records get a new sequence number.
</ParamField>

<ParamField path="company_id" type="Many2one (res.company)">
  The Odoo company associated with this request. Defaults to `self.env.company` (the currently active company).
</ParamField>

## Sequence

Bidding reference numbers are assigned by an Odoo sequence defined in `data/website_data.xml`:

```xml
<record id="seq_zonaweb_bidding_request" model="ir.sequence">
    <field name="name">Secuencia de Solicitudes de Licitación</field>
    <field name="code">zonaweb.bidding.request</field>
    <field name="prefix">BID/</field>
    <field name="padding">5</field>
    <field name="company_id" eval="False"/>
</record>
```

`company_id = False` means the sequence is global (shared across all Odoo companies in the database).

The sequence is called in the `create` override:

```python
@api.model_create_multi
def create(self, vals_list):
    for vals in vals_list:
        if vals.get('name', _('Nuevo')) == _('Nuevo'):
            vals['name'] = self.env['ir.sequence'].next_by_code(
                'zonaweb.bidding.request'
            )
    return super().create(vals_list)
```
