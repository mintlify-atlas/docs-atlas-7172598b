# Bidding request form

**URL:** `/licencias`  
**Template:** `zonaweb_website.bidding_page`  
**Form action:** `POST /submit-bidding`  
**Auth:** Public (no login required)  
**CSRF:** Required

The bidding (licitación) form allows any visitor to submit a service request or project proposal directly from the website. All submissions create a `zonaweb.bidding.request` record in the Odoo backend.

## Form fields

<ParamField path="company_name" type="string" required>
  Legal name of the requesting company.
</ParamField>

<ParamField path="contact_name" type="string" required>
  Full name of the primary contact person.
</ParamField>

<ParamField path="email" type="string" required>
  Contact email address. A confirmation email is sent to this address after submission.
</ParamField>

<ParamField path="phone" type="string">
  Contact phone number (optional).
</ParamField>

<ParamField path="service_type" type="selection" required>
  Type of service being requested. Options shown in the HTML form:

  | Value | Label |
  |---|---|
  | `software` | Desarrollo de Software |
  | `infrastructure` | Infraestructura de Redes |
  | `support` | Soporte Técnico |
  | `security` | Seguridad Informática |
  | `other` | Otro |

  <Note>
  The `consulting` selection value exists in the data model but is not exposed in the HTML form.
  </Note>
</ParamField>

<ParamField path="estimated_budget" type="float">
  Estimated project budget. The HTML input accepts decimal values (`step="0.01"`, `min="0"`). Defaults to `0.0` if left empty.
</ParamField>

<ParamField path="estimated_timeline" type="selection">
  Desired delivery timeline. Options shown in the HTML form:

  | Value | Label |
  |---|---|
  | `urgent` | Urgente (menos de 1 mes) |
  | `normal` | Normal (1-3 meses) |
  | `flexible` | Flexible (más de 3 meses) |
</ParamField>

<ParamField path="company_sector" type="selection">
  Industry sector of the requesting company. Options: retail, manufacturing, services, technology, healthcare, education, finance, other.
</ParamField>

<ParamField path="project_description" type="text" required>
  Detailed description of the project or service need.
</ParamField>

<ParamField path="specific_requirements" type="text">
  Any additional specific technical or functional requirements.
</ParamField>

## Submission flow

<Steps>
  <Step title="Visitor completes the form">
    The form at `/licencias` collects all required and optional fields. The layout is a two-column grid: the left column shows the 5-step process and a benefits panel; the right column contains the form card (`.zw-form`).
  </Step>
  <Step title="POST /submit-bidding is called">
    The form submits to `/submit-bidding` with CSRF token validation. The controller creates the `zonaweb.bidding.request` record using `sudo()` to bypass the public user's access restrictions.
  </Step>
  <Step title="Confirmation email is sent">
    The controller attempts to send the `email_template_bidding_confirmation` email to the submitter. Email failure is caught and logged as a warning — it does not block record creation.
  </Step>
  <Step title="Thank-you page is shown">
    On success, the visitor is rendered `zonaweb_website.bidding_thankyou`, which displays the assigned reference number (e.g., `BID/00001`) and a "Volver al Inicio" button.
  </Step>
  <Step title="Backend record is available">
    The new record appears in the Odoo backend in `draft` state, ready for internal review and approval.
  </Step>
</Steps>

## Error handling

If any unhandled exception occurs during record creation:
- The error is logged with `_logger.error`
- The visitor is shown the standard Odoo error page (`website.http_error`) with the message: "Error al procesar tu solicitud. Por favor, inténtalo de nuevo."

<Warning>
The form uses Odoo's built-in CSRF protection. Do not disable this in custom overrides — all POST submissions must include the CSRF token injected by `t-att-value="request.csrf_token()"`.
</Warning>

## Checking a submission status (API)

After submitting, you can programmatically check the status of a request using the JSON API:

```bash
curl -X POST https://your-odoo-domain.com/api/bidding-status \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc": "2.0", "method": "call", "params": {"bidding_id": "BID/00001"}}'
```

The endpoint returns a JSON object with `status` (the current state value) and `last_update` (a formatted datetime string). If the ID is not found it returns `{"error": "Licitación no encontrada"}`.

See [HTTP routes](/api/routes) for full API documentation.
